# AI Assistant Avito

## Стек технологий

- **Frontend:** React + TypeScript + MUI (Material UI)  
  - React даёт удобное построение компонентов и работу с состоянием.  
  - TypeScript добавляет типизацию и помогает ловить ошибки ещё на этапе разработки.  
  - MUI позволяет быстро и красиво оформлять интерфейс с готовыми компонентами (кнопки, карточки, таблицы и т.д.). 
  - Препроцессор sass, в частности scss в модульным подходом sass modules. 

- **Запросы к API:** Axios  
  - Axios удобен для отправки GET и PUT запросов на сервер.  

- **Состояния и кэширование:** React Query  
  - React Query позволяет легко загружать данные с сервера и кэшировать их.  
  - Поддерживает состояния загрузки, ошибки и повторные запросы.  

---

# Почему такой стек

- React + TypeScript — популярная связка, которая помогает быстро делать интерфейс и уменьшает количество ошибок.  
- MUI ускоряет некоторые моменты в создании элементов + удобная интеграция иконок.   
- Препроцессор Sass (SCSS) с модульным подходом (Sass Modules) — инкапсулирует стили компонентов, предотвращает конфликты имён, поддерживает переменные и миксины, а также позволяет безопасно использовать классы через TypeScript.
- Fastify лёгкий и быстрый, хорошо подходит для учебного проекта и небольшого API.  
- Axios + React Query делают работу с сервером простой и удобной.

# Инструкция по запуску проекта

## 1. Установка зависимостей

- Перейдите в корневую папку проекта и выполните:
  - `npm install`
  - или, если используете yarn: `yarn install`

## 2. Запуск фронтенда и бэкенда

- **Фронтенд** (React):
  - Запустите команду:
    - `npm start`
    - или `yarn start`
  - Приложение будет доступно обычно на [http://localhost:3000](http://localhost:3000)

- **Бэкенд** (Fastify):
  - Перейдите в папку бэкенда `server/`.
    - `cd server`
  - Запустите:
    - `npm start`
    - или `yarn dev`
  - Сервер поднимется на порту по умолчанию (например, [http://localhost:8080](http://localhost:8080) или как указано в настройках).

## 3. Настройка Ollama LLM

- **Установка Ollama:**
  - Скачайте Ollama с официального сайта: [https://ollama.com/](https://ollama.com/) и установите для вашей ОС.
  - После установки убедитесь, что команда `ollama` доступна в терминале.

- **Скачивание модели:**
  - Например, для скачивания модели `llama3.2` выполните:
    - `ollama pull llama3.2`
  - Можно использовать другую поддерживаемую модель по вашему выбору.

- **Запуск Ollama-сервера:**
  - Для запуска сервера просто выполните:
    - `ollama serve`
  - По умолчанию Ollama слушает на `http://localhost:11434`.

## 4. Настройка CORS для локального сервера

- Если фронтенд и бэкенд работают на разных портах, убедитесь, что CORS разрешён на сервере (Fastify).
  - В проекте уже настроен CORS, но если возникают ошибки, проверьте, что в настройках Fastify добавлен плагин cors:
    ```js
    fastify.register(require('@fastify/cors'), {
      origin: '*', // или укажите конкретный origin, например, 'http://localhost:3000'
    });
    ```
  - После изменений перезапустите сервер.

## 5. Проверка работоспособности AI-функций

- Убедитесь, что:
  - Ollama сервер работает (`ollama serve` не выдаёт ошибок).
  - Модель скачана и загружена.
  - Фронтенд и бэкенд успешно запущены.
- На сайте попробуйте воспользоваться AI-функциями (например, генерация описания или ответы на вопросы).
  - Если что-то не работает, проверьте консоль браузера и терминал сервера на наличие ошибок.
