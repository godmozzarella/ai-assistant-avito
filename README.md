# AI Assistant Avito

## Стек технологий

- **Frontend:** React + TypeScript + MUI (Material UI)  
  - React даёт удобное построение компонентов и работу с состоянием.  
  - TypeScript добавляет типизацию и помогает ловить ошибки ещё на этапе разработки.  
  - MUI позволяет быстро и красиво оформлять интерфейс с готовыми компонентами (кнопки, карточки, таблицы и т.д.).  

- **Запросы к API:** Axios  
  - Axios удобен для отправки GET и PUT запросов на сервер.  

- **Состояния и кэширование:** React Query  
  - React Query позволяет легко загружать данные с сервера и кэшировать их.  
  - Поддерживает состояния загрузки, ошибки и повторные запросы.  

---

# Почему такой стек

- React + TypeScript — популярная связка, которая помогает быстро делать интерфейс и уменьшает количество ошибок.  
- MUI ускоряет оформление приложения, не тратя много времени на CSS + есть опыт работы с ним.  
- Fastify лёгкий и быстрый, хорошо подходит для учебного проекта и небольшого API.  
- Axios + React Query делают работу с сервером простой и удобной.

# Что я сделал

- В сервисах я поправил возврат `id` при получении списка объявлений.  
  - Раньше API отдавал только сокращённые объекты без `id` для списка (`AdListItem`), и чтобы переходить на страницу детали, приходилось придумывать временный `derivedId`.  
  - Я добавил реальный `id` в список на сервере и в сервисах, чтобы фронтенд мог использовать настоящий уникальный идентификатор для карточки объявления.  
  - Теперь при клике на любую карточку можно безопасно перейти на `/ads/:id` и получить полные данные объявления с сервера.
